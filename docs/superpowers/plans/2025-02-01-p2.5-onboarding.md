# P2.5 首次使用引导 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将应用首次启动流程从当前的"协议→权限引导→主界面"替换为7步聊天式引导流程，在对话卡片中依次完成依赖安装、GitHub登录、主题选择、API配置和权限授权，提升首次体验的连贯性和完成率。
**Architecture:** 引导流程以聊天界面为载体，OnboardingViewModel管理步骤状态机（步骤0-7），每一步渲染为AI消息+交互卡片。步骤0在后台异步安装终端依赖，不阻塞用户操作；步骤1-6各对应一张功能卡片（GitHubLoginCard/ThemeSelectionCard/ApiKeyConfigCard/PermissionGrantCard）；步骤7展示完成消息后跳转主界面。引导状态通过SharedPreferences持久化，支持中断恢复。
**Tech Stack:** Kotlin, Jetpack Compose, SharedPreferences

## 任务1：创建 OnboardingPreferences 引导状态持久化

**文件：** `app/src/main/java/com/ai/assistance/operit/data/preferences/OnboardingPreferences.kt`

**步骤：**
- [ ] 创建 `OnboardingPreferences` 类，使用 SharedPreferences 存储引导状态
- [ ] 定义存储键：`onboarding_completed`（Boolean，引导是否完成）、`onboarding_current_step`（Int，当前步骤索引0-7）、`onboarding_step0_completed`（Boolean，依赖安装是否完成）、`onboarding_step2_completed`（Boolean，GitHub登录步骤是否完成/跳过）、`onboarding_step3_completed`（Boolean，主题选择是否完成）、`onboarding_step4_completed`（Boolean，DeepSeek API配置是否完成/跳过）、`onboarding_step5_completed`（Boolean，智谱API配置是否完成）、`onboarding_step6_completed`（Boolean，权限授权是否完成）
- [ ] 实现 `isOnboardingCompleted(): Boolean` 方法
- [ ] 实现 `getCurrentStep(): Int` 方法，默认返回0
- [ ] 实现 `setCurrentStep(step: Int)` 方法
- [ ] 实现 `isStepCompleted(step: Int): Boolean` 方法
- [ ] 实现 `setStepCompleted(step: Int, completed: Boolean)` 方法
- [ ] 实现 `setOnboardingCompleted(completed: Boolean)` 方法
- [ ] 实现 `resetOnboarding()` 方法，清除所有引导状态（用于调试）
- [ ] 使用单例模式 `companion object { getInstance(context) }`

**预期结果：**
```kotlin
class OnboardingPreferences private constructor(context: Context) {
    private val prefs = context.getSharedPreferences("onboarding_preferences", Context.MODE_PRIVATE)

    companion object {
        @Volatile private var INSTANCE: OnboardingPreferences? = null
        fun getInstance(context: Context): OnboardingPreferences {
            return INSTANCE ?: synchronized(this) {
                INSTANCE ?: OnboardingPreferences(context.applicationContext).also { INSTANCE = it }
            }
        }
        private const val KEY_ONBOARDING_COMPLETED = "onboarding_completed"
        private const val KEY_CURRENT_STEP = "onboarding_current_step"
        private const val KEY_STEP_COMPLETED_PREFIX = "onboarding_step_"
        private const val KEY_STEP_COMPLETED_SUFFIX = "_completed"
    }

    fun isOnboardingCompleted(): Boolean = prefs.getBoolean(KEY_ONBOARDING_COMPLETED, false)
    fun getCurrentStep(): Int = prefs.getInt(KEY_CURRENT_STEP, 0)
    fun setCurrentStep(step: Int) = prefs.edit().putInt(KEY_CURRENT_STEP, step).apply()
    fun isStepCompleted(step: Int): Boolean = prefs.getBoolean(KEY_STEP_COMPLETED_PREFIX + step + KEY_STEP_COMPLETED_SUFFIX, false)
    fun setStepCompleted(step: Int, completed: Boolean) = prefs.edit().putBoolean(KEY_STEP_COMPLETED_PREFIX + step + KEY_STEP_COMPLETED_SUFFIX, completed).apply()
    fun setOnboardingCompleted(completed: Boolean) = prefs.edit().putBoolean(KEY_ONBOARDING_COMPLETED, completed).apply()
    fun resetOnboarding() = prefs.edit().clear().apply()
}
```

**步骤：**
- [ ] 在 `requestPermission(permissionKey: String)` 方法中，根据权限类型构建对应的 Intent：

  - `overlay`：跳转 `Settings.ACTION_MANAGE_OVERLAY_PERMISSION` with data `Uri.parse("package:${context.packageName}")`

  - `location`：请求 `Manifest.permission.ACCESS_FINE_LOCATION` + `ACCESS_COARSE_LOCATION`
  - `notification`：Android 13+ 请求 `Manifest.permission.POST_NOTIFICATIONS`
  - `accessibility`：跳转 `Settings.ACTION_ACCESSIBILITY_SETTINGS`
- [ ] 通过事件流 `_permissionIntentEvent = MutableSharedFlow<Intent>()` 发送 Intent，OnboardingChatScreen 监听并启动 Activity
- [ ] 在 `checkPermissions()` 方法中，检查每项权限的实际状态：
  - `storage`：`Environment.isExternalStorageManager()` (API 30+) 或 `ContextCompat.checkSelfPermission()`
  - `overlay`：`Settings.canDrawOverlays(context)`
  - `battery`：`PowerManager.isIgnoringBatteryOptimizations(context.packageName)`
  - `location`：`ContextCompat.checkSelfPermission(ACCESS_FINE_LOCATION) == GRANTED`
  - `notification`：API 33+ 检查 `POST_NOTIFICATIONS` 权限
  - `accessibility`：检查无障碍服务是否启用
- [ ] 在 `OnboardingChatScreen` 中，使用 `LaunchedEffect` 监听 `permissionIntentEvent`，收到 Intent 后调用 `context.startActivity(intent)`
- [ ] 在 `OnboardingChatScreen` 的 `onResume` 生命周期中（通过 `LifecycleEventObserver`），调用 `viewModel.checkPermissions()` 刷新权限状态
- [ ] 当所有6项权限都已授权时，自动调用 `onStepCompleted(PERMISSION_GRANT)`

**预期结果：**
